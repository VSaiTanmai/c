SECTION 1: PROJECT OVERVIEW
================================================================================

CLIF (CLIF Log Investigation Framework) is an enterprise SIEM (Security
Information and Event Management) pipeline that ingests real system/network
logs, classifies them with VRL transforms, scores them with ML ensemble
models, hunts for threats with AI agents, verifies findings cryptographically,
and presents everything through a Next.js dashboard.

The system runs across TWO physical Windows PCs on the same LAN.
All infrastructure is Docker-containerized (25 containers total).

Previous sessions accomplished:
- IP migration from dev network (10.55.88.x) to production LAN (10.14.17.x)
- Full end-to-end pipeline test: 82,712 events ingested successfully
- VRL bottleneck optimization (Kafka when_full: block → drop_newest)
- Dashboard data feasibility audit: all 7 pages mapped to real data sources
- Comprehensive gap analysis documented in DASHBOARD_DATA_AUDIT.md

SECTION 3: DATA PIPELINE FLOW
================================================================================

Real Logs
  │
  ▼
Vector (VRL parse/classify, port 1514 syslog, 8687 HTTP)
  │  Single mega_transform: pre-classification fast path, 512-byte prefix
  │  regex (~9 ops/event), branches into 4 event types
  ▼
Redpanda (11 topics, 12 partitions each)
  │  Topics: raw-logs, security-events, process-events, network-events,
  │          triage-scores, anomaly-alerts, hunter-tasks, hunter-results,
  │          verifier-results, dead-letter, templated-logs
  │          (+ verifier-tasks with 6 partitions)
  ▼
Go Consumer ×2 (Kafka → ClickHouse batch writer)
  │  Batch: 200K events or 2s flush, LZ4 compression
  │  UUID5 deterministic event_id from topic:partition:offset
  ▼
ClickHouse (12 base tables + 14 materialized views)
  │
  ├──▶ Triage Agent ×4 (consume raw-logs, security/process/network-events)
  │      LightGBM + EIF + ARF ensemble, SHAP explainer, Drain3 templates
  │      Combined = lgbm×0.80 + eif×0.12 + arf×0.08
  │      Action: discard (<0.39) / monitor (0.39-0.89) / escalate (≥0.89)
  │      Escalated events → hunter-tasks + anomaly-alerts topics
  │      Output → triage_scores table
  │
  ├──▶ Hunter Agent (consume hunter-tasks)
  │      Secondary score gate: adjusted ≥ 0.70
  │      L1 parallel: Sigma rules, SPC, Graph, Temporal, LanceDB similarity
  │      L2 parallel: MITRE mapping, Campaign detection
  │      ML scoring: CatBoost or heuristic
  │      Output → hunter_investigations table + hunter-results topic
  │
  ├──▶ Verifier Agent (consume verifier-tasks)
  │      Verdict: true_positive / false_positive / inconclusive
  │      Evidence verification via Merkle tree integrity
  │      Priority: P1-P4, generates analyst_summary + report_narrative
  │      Output → verifier_results table
  │
  └──▶ Merkle Service (periodic daemon, every 30s, 1800s batch window)
         SHA256 hash chains over event batches per source table
         Stores merkle_root, leaf_hashes, s3_key → evidence_anchors table
         Archives to MinIO S3 (clif-evidence-archive bucket)

SECTION 5: AGENT SERVICES (all on PC2)
================================================================================

TRIAGE AGENT (Flask, ports 8300-8303, 4 replicas in consumer group)
  Input topics: raw-logs, security-events, process-events, network-events
  Output: triage-scores (all), anomaly-alerts + hunter-tasks (escalated)
  Models: LightGBM (0.80 weight) + Extended Isolation Forest (0.12) +
          Adaptive Random Forest (0.08)
  Features: Drain3 template mining, template_rarity, IOC matching
  Thresholds: suspicious ≥0.39, anomalous ≥0.89 (escalate)
  Batch: 500 msgs or 500ms, polling every 100ms
  Key endpoints: GET /health, GET /stats, GET /model/info
  Config file: agents/triage/config.py
  Logic: agents/triage/score_fusion.py (fusion + routing)
  App: agents/triage/app.py

HUNTER AGENT (FastAPI, port 8400, 1 replica)
  Input: hunter-tasks topic
  Output: hunter-results topic → hunter_investigations table
  Secondary gate: adjusted_score ≥ 0.70
  L1 parallel (5 tasks): Sigma rules, SPC engine, Graph builder,
                          Temporal correlator, LanceDB similarity
  L2 parallel: MITRE mapper, Campaign detector
  ML: CatBoost auto-switch or heuristic scoring
  Key endpoints: GET /health, GET /stats
  Config: agents/hunter/config.py
  Investigation modules: agents/hunter/investigation/
    campaign_detector.py, graph_builder.py, mitre_mapper.py,
    similarity_searcher.py, spc_engine.py, temporal_correlator.py

VERIFIER AGENT (FastAPI, port 8500, 1 replica)
  Input: verifier-tasks topic (from hunter)
  Output: verifier_results table
  Verdict: true_positive / false_positive / inconclusive
  Merkle integrity verification, timeline reconstruction
  Generates: analyst_summary, report_narrative, priority (P1-P4)
  Key endpoints: GET /health

XAI SERVICE (FastAPI, port 8200, 1 replica)
  SHAP explanations for individual events
  Key endpoints: GET /xai/status, GET /model/features,
                 POST /explain (event_id → SHAP values)
  App: agents/xai-service/app.py

MERKLE SERVICE (port 8600, daemon mode)
  Periodic: every 30s (MERKLE_DAEMON_INTERVAL)
  Batch window: 1800s (MERKLE_BATCH_WINDOW)
  SHA256 hash chains, stores to evidence_anchors + MinIO S3

LANCEDB SERVICE (port 8100)
  Semantic vector search with all-MiniLM-L6-v2 embeddings
  Syncs from ClickHouse every 30s, batch of 1000 events
  Used by Hunter similarity_searcher for event correlation

SECTION 12: KNOWN ISSUES & QUICK FIXES
================================================================================

1. DUAL PROCESS PORT 8200: Docker clif-ai-classifier binds 8200, conflicts
   with uvicorn XAI service. Fix: docker stop clif-ai-classifier after
   every docker compose up.

2. DOCKER PIPE NOT FOUND: Docker Desktop engine pipes not initialized.
   Fix: Restart Docker Desktop, wait 45s for pipes.

3. DASHBOARD ALL ZEROS: ClickHouse down or circuit breaker tripped.
   Fix: Check CH ping (localhost:8123), restart Docker if needed.
   Circuit breaker: dashboard/src/lib/clickhouse.ts (10s cooldown, threshold 3)
   Stale cache: dashboard/src/lib/cache.ts preserves last good data.

4. XAI SHAP ERROR: "only length-1 arrays" — SHAP >=0.50 API change.
   Fixed in ai-agents/inference/explainer.py (ndim==3 check).

5. AI SERVICE START: Kill all uvicorn → docker stop clif-ai-classifier →
   cd ai-agents && python -m uvicorn ai_service:app --host 0.0.0.0 --port 8200

6. NEXT.JS DEV: Must use `next dev` not `next start` for live changes.
   Port 3001. Clear .next cache if stale.
